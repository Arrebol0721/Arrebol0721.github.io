电子电路的数字抽象

发表于 2025-03-08 更新于 2025-03-13

经常在C++语言中使用逻辑运算符(与&&或||非！)和位运算符(&,|,^,~,<<,>>)，离不开数字电路的基本实现。物理器件和数学世界的关联最底层，在于对连续的物理数据进行了一种离散化处理，称之为数字抽象。

数字量和模拟量

数字电路：高集成处理用一个离散的电压序列表示的信息。
数字量：在时间上和数量上都是离散的不连续的。

模拟电路：高精度处理用连续的模拟电压/流值表示的信息。
模拟量：数字量以外的物理量。

电平和静态原则

高低电平实现数字信号

逻辑0：0.0V-2.5V
逻辑1：2.5V-5.0V

进一步，指定静止区域

逻辑0：0V-2V
逻辑1：3V-5V

最后，对发送者施加更严格的电压限制，而接收者的解释范围更宽。
噪声容限：对于给定逻辑值，指定的输出电压和相应的接收者禁止区域电压阈值之差的绝对值称为该逻辑值的噪声容限。

逻辑1的噪声容限：$ V_{OH}-V_{IH} $
逻辑0的噪声容限：$ V_{IL}-V_{OL} $

静态原则：是数字器件的规范。静态原则要求器件能够正确地解释输入阈值$(V_{IL}和V_{IH})$规定范围内的电压。如果为器件提供有效输入，该原则还要求器件能够产生满足阈值$(V_{OL}和V_{OH})$ 要求的有效输出电压

布尔逻辑

与，或，非，异或。

逻辑函数表示方式

  1. 真值表
  2. 逻辑式
  3. 逻辑图(组合门)
  4. 波形图
  5. 卡诺图（表示最小项的卡诺图化简）

逻辑函数标准形式

最小项之和：对应真值表：

  1. 在输入变量任意取值下，有且仅有一个最小项的值为1
  2. 全体最小项之和为1
  3. 任意两个最小项之积为0
  4. 两个相邻的最小项之和还可以合并，消去一对因子，只留下公共因子
    相邻：仅一个变量不同的最小项（化简重要性质）

简化逻辑表达

$A \cdot \overline{A}=0$
$A + \overline{A}=1$
$A+\overline{A}B=A+B$

摩根定理

$\overline{A \cdot B} = \overline{A}+\overline{B}$
$\overline{A+B}=\overline{A} \cdot \overline{B}$

每日1题-花园最大美丽值

发表于 2025-03-08

题目描述

Alice 是 n 个花园的园丁，她想通过种花，最大化她所有花园的总美丽值。
给你一个下标从 0 开始大小为 n 的整数数组 flowers ，其中 flowers[i] 是第 i 个花园里已经种的花的数目。已经种了的花不能移走。同时给你 newFlowers ，表示 Alice 额外可以种花的最大数目。同时给你的还有整数 target ，full 和 partial 。
如果一个花园有 至少 target 朵花，那么这个花园称为完善的，花园的总美丽值为以下分数之和 ：

  • 完善花园数目乘以 full.
  • 剩余不完善花园里，花的最少数目乘以 partial 。如果没有不完善花园，那么这一部分的值为 0 。
    请你返回 Alice 种最多 newFlowers 朵花以后，能得到的最大总美丽值。

题解

已知；

  • 当前每个花园种多少花的数组
  • 额外可以多种多少花
  • 花园最大种多少花的目标值
  • 还有一个系数partial

分析：
首先定义满足条件的花园个数fcnt，然后将花园数组排序，将满足条件的花园排除，如果全部已经满足条件，直接得出结果。
我们自己定义不完善花园最大可以种的花。
开始倒序遍历数组，首先如果全部花坛都不完善，求最大T。
如果倒序一个一个让花园完善，如果某次剩下可种的花为负数，直接break出循环。
注意循环不变量。j是每次循环不完善的花园数量，T是每次循环中不完善花园最大种花数量，sum是每次循环中未完善花园总的花数量

题目描述

给你一个下标从 0 开始的整数数组nums 。每次操作中，你可以：

  • 选择两个满足 0 <= i, j < nums.length 的不同下标 i 和 j 。
  • 选择一个非负整数 k ，满足 nums[i] 和 nums[j] 在二进制下的第 k 位（下标编号从 0 开始）是 1 。
  • 将 nums[i] 和 nums[j] 都减去 2的k次方 。

如果一个子数组内执行上述操作若干次后，该子数组可以变成一个全为 0 的数组，那么我们称它是一个美丽的子数组。
请你返回数组 nums 中美丽子数组的数目。

题解

例如{4,3,1,2,4}:
100
011
001
010
100
每一轮操作可以消去某一列两个1，能把所有数变成0，那么这个子数组就是美丽子数组。
我们结合异或的性质，不同为1，相同为0，如果这几个数异或值为0，那么就是美丽子数组。
可以利用前辍和还有哈希表来计算一共有多少连续子数组。

设置哈希表默认值为{0,1}是在查找前面元素的前辍和的时候。前零个元素的前辍和也应该算上。

每日1题-破坏字符串

发表于 2025-03-06

题目描述

给你一个由小写英文字母组成的回文字符串palindrome。请你将其中一个字符用任意小写英文字母替换，使得结果字符串的字典序最小，且不是回文串。
请你返回结果字符串。如果无法做到，则返回一个空串。
如果两个字符串长度相同，那么字符串 a 字典序比字符串 b 小可以这样定义：在 a 和 b 出现不同的第一个位置上，字符串 a 中的字符严格小于 b 中的对应字符。例如，”abcc” 字典序比 “abcd” 小，因为不同的第一个位置是在第四个字符，显然’c’比’d’小。

题解

给的一定是回文串。那么修改成’a’字典序变得最小。
三种情况：

  1. 如果字符串只有一个字符，无论怎么修改都是回文串。
  2. 如果字符串有不为 a 的字符，把第一个遇见的字符修改为 ‘a’ 字典序最小.
  3. 如果字符串全部为a，那么修改最后一个字符为’b’

《软件测试》笔记1

发表于 2025-03-05 更新于 2025-03-11

今天跟面试官交流中依旧聊到了我对软件测试的理解是什么。
但是只是笼统的讲了一下发现bug，追踪bug，并没有想起来我曾经翻过这本书。
前人栽树，后人并不能体会感受到树的生长过程中遇到的种种问题，只能通过这种形式记录理解。

这本书首先探讨了测试背景概述，然后将测试分为黑盒白盒两类。
重点在于动态黑盒测试，其又分为数据测试和状态测试。

软件测试究竟要做什么

尽可能早的找出缺陷，并确保其得已修复
对软件测试的理解实际上有很大商业成分，关系到公司利润等；也可以从软件本身的角度去分析，关系到软件的完善性。

什么是缺陷

实际上软件缺陷不只是代码bug，还包括种种：

  1. 产品说明书指明或者未明确指明的功能。
  2. 产品说明书指明或者未明确指明的错误。
  3. 软件测试员认为最终用户认为不好。难以理解，不易使用的点。

测试分类

黑盒测试只知道程序做了什么，不知道为什么会这样，软件如何运行。
白盒测试可以访问程序员的代码。

静态黑盒测试

指测试产品说明书，并在软件编写之前找出问题。

动态黑盒测试

拿到产品说明书或者需求文档，定义用例，选择用例，开始动态黑盒测试。
等价类划分：选择测试用例的方法是等价划分。一个等价类是值测试相同目标或者暴露相同软件缺陷的一组测试用例。如果为了减少测试用例的数量过度划分等价类，就有可能漏掉那些可能暴露缺陷的测试。
数据测试：对软件的最简单认识就是两部分，数据和程序。数据包括键盘输入，鼠标单击，磁盘文件，打印输出。程序指可执行的流程，转换，逻辑，运算，根据一些关键的原则进行等价类划分，这些原则包括边界条件，次边界条件，空值和无效数据。

  1. 边界条件：指软件运行在计划操作界限的边界情况。计划是一个很好的形容。
  2. 次边界条件：隐藏的边界条件，最终用户几乎看不到。跟代码有关系，例如ASCII表。
  3. 默认，空白，空值，零值和无：很容易被忽视，一定要单独建立等价类
  4. 非法，错误，不正确，垃圾数据：很有意思也很有意义，用户量大了之后一定有用户错误使用，尽情试验吧。

状态测试：目前为止测试的是数据————数字，文字，软件输入和输出。另一方面就是通过不同的状态验证程序的逻辑流程，例如画图的程序提供的选项，一旦选中某项，改变了外观，菜单或者某些操作，必须测试软件或者程序的状态及其转换。可以通过建立状态转换图来测试。
失败状态测试：以上都是通过性测试，相反的做法是找到是测试软件失败的案例，此类案例有竞争条件，重复，压迫，和重负。好抽象的概念！竞争条件案例有同时打开关闭多个软件，同时访问数据库，同时共享一台外围设备。重复打开关闭软件能找出内存泄漏。压迫测试是让软件在不理想情况运行，限制内存CPU速度等。重负是让软件尽量处理尽可能大的数据，最大限度运行软件。
其他黑盒测试技术：像笨拙的用户一样思考，在已经找到缺陷的地方再找找，像黑客一样思考，凭借直觉经验预感。

静态白盒测试

静态白盒测试指不执行软件的情况下有条理的审查软件设计，体系结构和代码，从而找出软件缺陷，也称为结构化分析。
包含编码标准和规范，还有各种代码错误，就是我们常说的代码审查。

  1. 数据引用错误
  2. 数据声明错误
  3. 计算错误
  4. 比较错误
  5. 控制流程错误
  6. 子程序参数错误
  7. 输入输出错误

动态白盒测试

是指在看到软件的工作方式时，根据获得的信息对软件进行测试。

  • 单元测试和集成测试
  • 数据覆盖
  • 代码覆盖

后续

还有配置测试，兼容性测试，易用性测试，外国语言测试，安全性测试等大类。

接口

接口的概念

接口不是类，而是对希望符合这个接口的类的一组需求。
例如，Arrays的sort方法可以对对象数组进行排序，但是对象所属的类必须实现Comparable接口：

任何实现Comparable接口的类都需要包含compareTo方法，接口中的方法默认是public abstract的。

接口的实现

  1. 将类声明为实现给定的接口
  1. 对接口中的所有方法提供定义

接口的使用

无法实例接口，但是可以定义接口变量，接口变量必须引用实现了这个接口的对象。
可以使用instanceof检查一个对象是否实现了某个接口。
接口也可以扩展，从通用性较高的接口扩展到专用性较高的接口，Java的标准库对此有应用。

继承

子类

Java使用extends代替了C++中的冒号（：），在Java中，所有继承都是公共继承。

覆盖方法

需要使用super关键字来调用父类的方法访问父类的私有字段

子类构造器

由于子类无法访问父类的私有字段，所以通过super关键字访问父类构造方法，如果子类的构造器没有显式调用父类的构造器，会默认调用父类的无参数构造器
在C++中，会使用初始化列表来调用父类的构造器

多态和动态绑定

一个对象变量可以指示多种实际类型的的现象称为多态，在运行时能够自动选择适当的方法称为动态绑定。
在C++中实现多态需要需要将成员函数声明为virtual，而Java中多态是默认的，如果不希望让一个方法是虚拟的，可以将它标记为final

强制类型转换

尽量不使用，如果使用，只能在继承层次内使用，且将父类变为子类时要使用instanceof检查

抽象类

在C++中，有一种抽象方法称为纯虚函数，要在末尾用=0标记，如果至少有一个纯虚函数，这个C++类就是抽象类，在C++中没有提供用于表示抽象类的关键词。
在Java中即使不含抽象方法也可以将类声明为抽象类，只需要使用abstract关键词。
注意，抽象类不能实例化，可以定义一个抽象类的对象变量，但只能引用非抽象子类的对象。

受保护访问

4种修饰符

  1. private-仅对本类可见
  2. 默认-对本包可见
  3. protected-对本包和所有子类可见
  4. public-对外部完全可见

Object：所有类的超类

Object类中equals方法用于检测一个对象是否等于另一个对象，比较两个对象的引用是否相同

如果x.object(y);中有一个null引用，会报错。
Java提供了Object.equals（a，b）静态方法可以防备比较对象为null。
如果需要按内容比较，经常需要重写equals方法，例如String类和基础类型包装类都重写了equals方法。
Object类还有一个toString方法，默认返回对象的类名和哈希码，建议重写该方法来返回更有意义的字符串。

泛型数组列表

ArrayList是一个有类型参数的泛型类。
声明数组列表甚至可以有三种写法：

  1. ArrayList<Employee> staff=new ArrayList<Employee>();
  2. var staff=new ArrayList<Employee>();
  3. ArrayList<Employee> staff=new ArrayList<>();

最后一种方法，编译器会检查并自动填写。
ArratList类似于C++的vecter模板，都是泛型类型，只是vector重载了[]运算符从而便于访问，还有a=b在vector是按值拷贝，在Java中是让a跟b引用同一个数组列表。
由于没有重载运算符，我们需要通过get和set方法来访问元素，使用size来统计元素个数。

包装器类

如果想要定义一个整型数组列表，必须使用ArrayList,因为尖括号内不允许使用基本类型。
包装器有一种编译器负责的自动装箱特性：
list.add(3);会自动变换成list.add(Integer.valueOf(3));
int n=list.get(i);会自动转换成int n=list.get(i).intValue();
基本类型与对象包装器有一点要注意，==运算符应用于包装器对象检测的是是否有相同内存位置，采用equals方法可以解决这个问题。
