接口

接口的概念

接口不是类，而是对希望符合这个接口的类的一组需求。
例如，Arrays的sort方法可以对对象数组进行排序，但是对象所属的类必须实现Comparable接口：

任何实现Comparable接口的类都需要包含compareTo方法，接口中的方法默认是public abstract的。

接口的实现

  1. 将类声明为实现给定的接口
  1. 对接口中的所有方法提供定义

接口的使用

无法实例接口，但是可以定义接口变量，接口变量必须引用实现了这个接口的对象。
可以使用instanceof检查一个对象是否实现了某个接口。
接口也可以扩展，从通用性较高的接口扩展到专用性较高的接口，Java的标准库对此有应用。
通过接口调用方法，实现多态。

接口和抽象类

C++允许一个类有多个超类，称为多重继承。
但是Java为了避免多重继承的复杂性，规定每个类只能扩展一个类，不能实现多重继承。
但是每个类可以实现多个接口。

接口的应用

JDBC中的 Connection、Statement 就是接口，定义了数据库操作的规范。

继承

子类

Java使用extends代替了C++中的冒号（：），在Java中，所有继承都是公共继承。

覆盖方法

需要使用super关键字来调用父类的方法访问父类的私有字段

子类构造器

由于子类无法访问父类的私有字段，所以通过super关键字访问父类构造方法，如果子类的构造器没有显式调用父类的构造器，会默认调用父类的无参数构造器
在C++中，会使用初始化列表来调用父类的构造器

多态和动态绑定

一个对象变量可以指示多种实际类型的的现象称为多态，在运行时能够自动选择适当的方法称为动态绑定。
在C++中实现多态需要需要将成员函数声明为virtual，而Java中多态是默认的，如果不希望让一个方法是虚拟的，可以将它标记为final

强制类型转换

尽量不使用，如果使用，只能在继承层次内使用，且将父类变为子类时要使用instanceof检查

抽象类

在C++中，有一种抽象方法称为纯虚函数，要在末尾用=0标记，如果至少有一个纯虚函数，这个C++类就是抽象类，在C++中没有提供用于表示抽象类的关键词。
在Java中即使不含抽象方法也可以将类声明为抽象类，只需要使用abstract关键词。
注意，抽象类不能实例化，可以定义一个抽象类的对象变量，但只能引用非抽象子类的对象。

受保护访问

4种修饰符

  1. private-仅对本类可见
  2. 默认-对本包可见
  3. protected-对本包和所有子类可见
  4. public-对外部完全可见

Object：所有类的超类

Object类中equals方法用于检测一个对象是否等于另一个对象，比较两个对象的引用是否相同

如果x.object(y);中有一个null引用，会报错。
Java提供了Object.equals（a，b）静态方法可以防备比较对象为null。
如果需要按内容比较，经常需要重写equals方法，例如String类和基础类型包装类都重写了equals方法。
Object类还有一个toString方法，默认返回对象的类名和哈希码，建议重写该方法来返回更有意义的字符串。

泛型数组列表

ArrayList是一个有类型参数的泛型类。
声明数组列表甚至可以有三种写法：

  1. ArrayList<Employee> staff=new ArrayList<Employee>();
  2. var staff=new ArrayList<Employee>();
  3. ArrayList<Employee> staff=new ArrayList<>();

最后一种方法，编译器会检查并自动填写。
ArratList类似于C++的vecter模板，都是泛型类型，只是vector重载了[]运算符从而便于访问，还有a=b在vector是按值拷贝，在Java中是让a跟b引用同一个数组列表。
由于没有重载运算符，我们需要通过get和set方法来访问元素，使用size来统计元素个数。

包装器类

如果想要定义一个整型数组列表，必须使用ArrayList,因为尖括号内不允许使用基本类型。
包装器有一种编译器负责的自动装箱特性：
list.add(3);会自动变换成list.add(Integer.valueOf(3));
int n=list.get(i);会自动转换成int n=list.get(i).intValue();
基本类型与对象包装器有一点要注意，==运算符应用于包装器对象检测的是是否有相同内存位置，采用equals方法可以解决这个问题。

枚举类

枚举声明定义的类型是一个类，它刚好有固定个实例，不可构造新对象，可以用==比较。
每个枚举类都有一个静态的values方法，返回一个包含全部值的数组。

反射

不需要为其他Java程序员构建工具，暂时跳过。

继承的设计技巧

  1. 将公共操作和字段放在父类中
  2. 不要使用受保护的字段
  3. 使用继承实现is-a关系
  4. 使用多态，而不要使用类型信息
  5. 除非所有使用继承的方法都有意义，否则不要使用继承

java类与C++

多个源文件的使用

将Employee类存放在Employee.java中，将EmployeeTest类存放在EmployeeTest.java中，如果EmployeeTest类用到了Employee类，有两种编译文件的方法：

  • 使用通配符javac Employee*.java，编译所有匹配文件
  • javac EmployeeTest.java这种方式没有显式编译Employee.java，但是它会搜索Employee.java，对其进行编译；

Java构造函数

总是伴随new操作符使用,例如Employee number007("james",23)这种写法在C++中可以，在Java中不行

this

这个方法有一个隐式参数salary，可以在其前面加this
C++中，通常在类外面定义方法，在内部定义方法将自动成为内联方法，但是在Java中，所有方法必须在类内定义，是否为内联方法由Java虚拟机负责

基于类的访问权限

方法可以调用所属类的所有对象的私有数据
私有方法的重点在于类的设计者确保其不会在别处使用，可以随意删除它，而公共的就不可以简单删除掉

final关键字

  • 用在引用类型上只是表示存储在变量的对象引用不会再指向另一个不同的对象，不过这个对象可以更改。
  • 用在基本类型，该变量内容不可变

静态

  • 静态字段，即使没有对象，该字段也存在，例如Math类中的PI，可以用Math.PI访问这个常量，还有System.out。
  • 静态方法，不在对象上执行的方法，不可以访问实例字段，但是可以访问静态字段
  • 静态的历史问题，起初C语言使用静态来表示退出一个块后依然存在的局部变量，后来有了第二种含义，表示不能从其他文件访问的全局变量和函数。C++为了不引入新的关键字，第三次重用这个关键字，表示属于类而不属于任何类对象的变量和函数，这个含义与Java中这个关键字含义相同

方法参数

Java总是采用按值传递，C++中既有按值传递也有按引用传递（&）
java方法对参数的操作：

  • 方法不能修改基本数据类型的参数
  • 方法可以改变对象参数的状态
  • 方法不能让一个对象参数引用一个新的对象

对象构造

  • 重载
  • 默认字段初始化，布尔类型为false，引用为null，数值类型为0
  • 没有编写构造函数时，会提供一个默认构造，这个构造将所有字段设置为默认值，如果提供了有参构造，默认构造就不会提供
  • 显式字段初始化，在类定义中直接为字段赋值
  • 可以用this调用另一个构造方法

类的导入

  • 一般使用import java.time.LocalDate导入
  • 可以在使用类的时候使用完全限定名，例如java.time.LocalDate today=java.time.LocalDate.now()
  • 默认导入了java.lang.*，只能使用*号导入一个包中所有类

静态导入

import static java.lang.System.out
import static java.lang.System.*
导入后不必加类名前缀

在包中增加类

  • 使用package关键字，例如package com.horst.corejava;
  • 将源文件放在与完整包名匹配的子目录中，否则编译运行容易出错

包访问

public允许任何类使用，private允许定义他们的类使用，没有指定public或private，可以被同一个包中所有方法使用

类路径

JAR文件包含多个压缩形式的类文件和子目录。
类路径包含基目录，JAR文件，当前目录（.）

Java核心

  • 简单性：改进C++，剔除部分包括，头文件，指针，操作符重载，虚基类，结构？联合？
  • 面向对象：与C++主要不同在于多重继承，取而代之是接口
  • 分布式：Java有一个例程库，处理HTTP和FTP之类的TCP/IP协议
  • 健壮性：强调早期问题检测，后期动态运行检测代码问题
  • 安全性：防范运行时堆栈溢出，破坏进程空间之外的内存，未经授权读写文件
  • 体系结构中立：生成一个体系结构中立的目标文件格式，只要有Java运行时系统，实现跨平台运行
  • 可移植性：基本数据类型的大小以及有关运算都有明确说明，例如int固定32位，不像C++
  • 解释型：Java解释器可以在任何移植了解释器的机器上直接运行Java字节码
  • 高性能：字节码可以在运行时动态转换成对应运行这应用的特定CPU的机器码。
  • 多线程：处理器发展带来并发性。
  • 动态性：为运行的程序增加代码，库中可以自由添加新方法和实例变量，而对客户端没有影响

每日1题-设计一个文本编辑器

发表于 2025-02-27

题目描述

请你设计一个带光标的文本编辑器，它可以实现以下功能：

  • 添加：在光标所在处添加文本。
  • 删除：在光标所在处删除文本（模拟键盘的删除键）。
  • 移动：将光标往左或者往右移动。

当删除文本时，只有光标左边的字符会被删除。光标会留在文本内，也就是说任意时候 0 <= cursor.position <= currentText.length 都成立。

请你实现 TextEditor 类：

  • TextEditor() 用空文本初始化对象。
  • void addText(string text) 将 text 添加到光标所在位置。添加完后光标在 text 的右边。
  • int deleteText(int k) 删除光标左边 k 个字符。返回实际删除的字符数目。
  • string cursorLeft(int k) 将光标向左移动 k 次。返回移动后光标左边 min(10, len) 个字符，其中 len 是光标左边的字符数目。
  • string cursorRight(int k) 将光标向右移动 k 次。返回移动后光标左边 min(10, len) 个字符，其中 len 是光标左边的字符数目。

题解

创建两个栈，光标左右移动相当于两个栈的栈顶数据互相移动，插入删除相当于在左边那个栈上入栈出栈。
