Servlet 的作用

  • Java Servlet 是运行在 Web 服务器或应用服务器上的程序，它是作为来自 Web 浏览器或其他 HTTP 客户端的请求和 HTTP 服务器上的数据库或应用程序之间的中间层。

Servlet 基本使用

  • 需要继承 HttpServlet 的父类才能具备接受前台请求功能

    • 重写 doGet 和 doPost 方法
    • 这两个方法分别对应前台表单的 method 属性
    • 在方法中通过 getParameter 方法和 name 属性来获取前台数据

  • Servlet 类必须注册后才能使用

    • 注册信息中的<url-pattern>对应前台 action 属性

    • 做映射对应前端，对应后端地址

  • Servlet 3.0+ 支持通过注解（如 @WebServlet）配置，无需 web.xml:

    Spring Boot 等框架通常完全省略 web.xml，改用 Java 配置或自动装配.

Servlet 中的响应操作

  • 通过 resp 响应对象获取输出流 PrintWriter 对象

Servlet 的创建

  • 继承 HttpServlet
  • 继承 HttpServlet 的父类 GenericServlet
  • GenericServlet 类重写了 service 方法，根据请求方法的不同，将请求分发给 doGet 和 doPost 方法
  • 继承自 Servlet 方法需要重写init()，service()，getServletConfig()，getServletInfo()，destroy()5 个方法，展现了 servlet 对象的生命周期。第一次访问时创建对象，初始化对象，请求一次就会执行一次service()方法，在 Tomcat 关闭时会执行destroy()，
  • 多个 HTTP 请求到达 Servlet 容器产生一个实例，多线程交给service()处理，单例多线程

Servlet Config 对象

  • ServletConfig 对象，获取属于 Servlet 对象的配置项。通过 web.xml 中添加标签，在doGet()或者doPost()方法中都可以创建 ServletConfig 对象来获取中的配置信息

重定向和请求转发

请求转发是一种服务器内部的跳转机制，我们知道，重定向会使得浏览器去重新请求一个页面，而请求转发则是服务器内部进行跳转，它的目的是，直接将本次请求转发给其他 Servlet 进行处理，并由其他 Servlet 来返回结果，因此它是在进行内部的转发。

请求转发可以携带数据。

  • 重定向是两次请求，请求转发是一次请求
  • 请求转发地址栏不会发生改变， 重定向地址栏会发生改变
  • 请求转发可以共享请求参数 ，重定向之后，就获取不了共享参数了
  • 请求转发只能转发给内部的 Servlet

ServletContext 对象

ServletContext 全局唯一，它是属于整个 Web 应用程序的，我们可以通过getServletContext()来获取到此对象。

ServletContext 对象，可以用来获取 web.xml 中的全局配置项，使用方法和 ServletConfig 对象一样。

每日1题-转换二维数组

发表于 2025-03-19 更新于 2025-03-20

题目描述

给你一个整数数组 nums 。请你创建一个满足以下条件的二维数组：

  • 二维数组应该只包含数组 nums 中的元素。
  • 二维数组中的每一行都包含不同的整数。
  • 二维数组的行数应尽可能少 。
    返回结果数组。如果存在多种答案，则返回其中任何一种。
    请注意，二维数组的每一行上可以存在不同数量的元素。

题解

通过哈希表先统计所有元素出现次数，然后遍历哈希表的同时将元素出现次数减1，如果次数减到了0，就擦除元素。
注意C++的迭代器擦除元素函数的使用规则。

机考题目-解密犯罪时间

发表于 2025-03-15

题目描述

小王在进行游戏大闯关，有一个关卡需要输入一个密码才能通过，密码获得的条件如下：
在一个密码本中，每一页都有一个由 26 个小写字母组成的若干位密码，每一页的密码不同，需要从这个密码本中寻找这样一个最长的密码，从它的末尾开始依次去掉一位得到的新密码也在密码本中存在。
请输出符合要求的密码，如果有多个符合要求的密码，则返回字典序最大的密码。
若没有符合要求的密码，则返回空字符串

输入描述

密码本由一个字符串数组组成，不同元素之间使用空格隔开，每一个元素代表密码本每一页的密码。

输出描述

一个字符串。

用例输入

h he hel hell hello

用例输出

hello

题解

采用哈希集合加快查找。对数组元素排序，先按照长度排序，再按照字典序排序。
然后对每个元素查找它的所有符合条件的子字符串是否在数组中。例如hello的符合条件的子串有hell，hel，he，h。
找到的第一个字符串就是答案。

机考题目-TLV编码

发表于 2025-03-15

题目描述

TLV 编码是按 Tag Length Value 格式进行编码的。
一段码流中的信元用 tag 标识，tag 在码流中唯一不重复，length 表示信元 value 的长度，value 表示信元的值，码流以某信元的 tag 开头，tag 固定占一个字节，length 固定占两个字节，字节序为小端序。
现给定 TLV 格式编码的码流以及需要解码的信元 tag，请输出该信元的 value。
输入码流的 16 进制字符中，不包括小写字母；且要求输出的 16 进制字符串中也不要包含小写字母；码流字符串的最大长度不超过 50000 个字节。

输入描述

第一行为第一个字符串，表示待解码信元的 tag；
第二行为一个字符串，表示待解码的 16 进制码流；每个字节之间用空格分割。

输出描述

输出一个字符串，表示待解码信元以 16 进制表示的 value。

用例输入

31
32 01 00 AE 90 02 00 01 02 30 03 00 AB 32 31 31 02 00 32 33 33 01 00 CC

用例输出

32 33

用例分析

首先题意是这样的，第一行输入一个tag标识表示要输出对应信元的value值。
第二行输入一串字节流，表示待解码的码流。注意length的字节序为小端序。
例如用例要求找到31对应信元的value值。
第一段信元的tag为32，长度为01 00，换算后为1，表示后面的value值为AE；
第二段信元的tag为90，长度为02 00，换算后为2，表示后面的value值为01 02；
第三段信元的tag为30，长度为03 00，表示后面的value值为AB 32 31；
第四段信元的tag为31，长度为02 00，表示后面的value值为32 33；

题解

解答这道题我大量应用了c++的流库sstream，将字符串转换成流处理。
包括分割字符串，将字符串以16进制转换成整数。

那成员函数跟静态函数有什么区别

在访问权限上，调用方式和this指针上有区别。

《人月神话》笔记1

发表于 2025-03-13

面试的时候经常被问自己对加班的看法是什么？
记得某一次看到别人推荐这本书，讲了一些软件开发的困境，没想到大头还是项目管理，想起来第一份工作的项目管理内容也很多，早上开会，晚上开会，觉得很烦。后来去翻看了才理解了一些工作中的项目管理。

人月神话

这本书的名字是一种项目开发的误区，把（1人/月）当成一个简单的衡量单位，1个人工作2个月和2个人工作一个月是等效的，可以换算的。例如预估12人/月的开发项目，原本有3个人，只需要开发4个月就能完成，但是中间发现无法按时完成，准备增加人手的时候，简单的把后面进来的人按照（人/月）来换算而没有考虑人员之间的沟通，会导致项目更加落后。当然还有任务的分配，新人员的培训，新的系统测试等等因素。不断加人只会导致项目像焦油坑一样淹死大型动物，很形象的比喻。
那么最好的解决办法是什么？不添加任何人手，让现有的人手加班，同时保持高效率加班。
这样客观来看的话，加班就很合理了。

沟通的负担

影响项目进度的最大因素是人员之间的沟通，确实，人越多结构越复杂，沟通的量会指数级增长。
三种沟通途径：私下沟通，会议，文档。
由此说明了文档的重要性，良好，实时更新的文档和文档结构确实可以很有效的减少沟通时间成本。
团队的组织结构也是为了较少必要的交流和沟通。

安排

书中将软件任务的进度是这样安排的：
1/3 计划
1/6 编码
1/4 构件测试和早期系统测试
1/4 系统测试。所有的构建已完成
这样的安排狠狠强调了计划和测试的重要性。
由于先后开发顺序的影响，系统测试和单元测试与项目进度牵扯的最深，而且它的时间还受到bug数量和难以捕捉的程度。

贯彻执行与测试

为了保证贯彻执行这一原则，测试小组很有必要，因为测试总会发现一些没有贯彻执行，设计决策没有准确实现或者理解正确的地方。

概念完整性

概念完整性是系统设计中最重要的概念。
这个应该是由于软件开发的概念功能多样，导致功能和理解上的复杂程度的比值才是系统设计的最终测试标准。
为了促进开发和测试的明确性。

电子电路的基本器件

发表于 2025-03-13 更新于 2025-03-15

半导体

纯硅：电的不良导体。
N型硅：添加第五组元素（例如磷）到硅中，得到一种许多未被束缚在共价键中的自由电子的晶体。
P型硅：添加第三组元素（例如铝）到硅中，而是产生的晶体在存在大量电子空额，我们称之为空穴。

二极管

如果用一个电池使二极管的P区相对于N区为正，正向偏置，电流通过。
如果二极管的N区相对于P区为正，反向偏置，几乎没有电流。
二极管的伏安特性：电流在电压达到0.6V左右时迅速上升，因此在毫安尺度下硅二极管有一个0.6V-0.7V的阈值。

三极管(BJT)

NPN型；箭头B指向E
PNP型：箭头E指向B
原理：基极(B)小电流控制集电极(C)和发射极(E)大电流。

MOS管

原理：漏源极进出。MOSFET的工作类似源极和漏极之间的一个开关，当栅极(G)电压超过阙值时导电沟道形成，开关开通。
金属层：金属层被氧化物层（厚度不成比例）分割开来，这样就确保了金属层不会与该元件的其他部分接触。
SR模型：MOSFET 形成的导电 N 沟道并非理想导体，具有一定的电阻R。同时注意到该电阻与沟道的几何尺寸有关。设沟道长度（栅极长度）为 L，沟道宽度为 W。则电阻正比于L/W。

CMOS管

概述：反相器，将输入电平取反，通过NMOS管和PMOS管组合形成互补MOS。
结构：栅极连接在一起作为输入，漏极连接在一起作为输出。NMOS管接地，PMOS管接电源。
工作：输入高电平，NMOS管导通，输出低电平；输入低电平，PMOS管导通，输出高电平。
静态特性：输入信号不变化时，功耗低。

每日1题-统计最大元素出现至少K次的子数组

发表于 2025-03-12

题目描述

给你一个整数数组 nums 和一个正整数 k 。
请你统计有多少满足「 nums 中的最大元素」至少出现 k 次的子数组，并返回满足这一条件的子数组的数目。
子数组是数组中的一个连续元素序列。

题解

此题是典型的滑动窗口。
首先找到nums中的最大元素，然后滑动窗口保证nums中最大元素超过k个。
例如[1,3,2,3,3],遍历到第二个3的时候，cnt_mx等于k了，移动左端点直到区间内最大元素个数不超过k，此时，左端点位于第一个2处，left=2，此时[1,3,2,3][3,2,3]都满足条件，把答案增加left。当右端点移动到末尾元素时，左端点也移动到末尾元素，此时[1,3,2,3,3][3,2,3,3][2,3,3][3,3]都满足条件，答案增加left=4。
