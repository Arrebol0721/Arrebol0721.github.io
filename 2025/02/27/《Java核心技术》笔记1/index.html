Java核心

  • 简单性：改进C++，剔除部分包括，头文件，指针，操作符重载，虚基类，结构？联合？
  • 面向对象：与C++主要不同在于多重继承，取而代之是接口
  • 分布式：Java有一个例程库，处理HTTP和FTP之类的TCP/IP协议
  • 健壮性：强调早期问题检测，后期动态运行检测代码问题
  • 安全性：防范运行时堆栈溢出，破坏进程空间之外的内存，未经授权读写文件
  • 体系结构中立：生成一个体系结构中立的目标文件格式，只要有Java运行时系统，实现跨平台运行
  • 可移植性：基本数据类型的大小以及有关运算都有明确说明，例如int固定32位，不像C++
  • 解释型：Java解释器可以在任何移植了解释器的机器上直接运行Java字节码
  • 高性能：字节码可以在运行时动态转换成对应运行这应用的特定CPU的机器码。
  • 多线程：处理器发展带来并发性。
  • 动态性：为运行的程序增加代码，库中可以自由添加新方法和实例变量，而对客户端没有影响

后续阅读可以多结合这11点

从一个基本的代码开始分析

  1. 首先Java区分大小写
  2. 规范，类名首字母大写，文件名与类名同名，并以.java作为扩展
  3. main方法必须为public
  4. Java中所有函数都是某个类的方法，main必须是静态的

基本数据类型

8中基本类型，4种整数，2种浮点数，1种字符类型char和一种boolean类型

  • 整数包括int（4字节），short（2字节），long（8字节），byte（1字节），没有无符号类型
  • 浮点数有float（4字节）和double（8字节）类型
  • char类型（16位），单引号，Unicode历史问题。
  • boolean类型，区分C++，0不能代替false，非零不能代替true。

变量与常量

  • 声明变量后必须初始化才能使用
  • 常量使用final关键字，只能被赋值一次，不能再更改；类常量使用static final；如果使用public，其他类也可以使用
