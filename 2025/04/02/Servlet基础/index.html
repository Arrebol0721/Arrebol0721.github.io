Servlet基础

发表于 2025-04-02

Servlet 的作用

  • Java Servlet 是运行在 Web 服务器或应用服务器上的程序，它是作为来自 Web 浏览器或其他 HTTP 客户端的请求和 HTTP 服务器上的数据库或应用程序之间的中间层。

Servlet 基本使用

  • 需要继承 HttpServlet 的父类才能具备接受前台请求功能

    • 重写 doGet 和 doPost 方法
    • 这两个方法分别对应前台表单的 method 属性
    • 在方法中通过 getParameter 方法和 name 属性来获取前台数据

  • Servlet 类必须注册后才能使用

    • 注册信息中的<url-pattern>对应前台 action 属性

    • 做映射对应前端，对应后端地址

  • Servlet 3.0+ 支持通过注解（如 @WebServlet）配置，无需 web.xml:

    Spring Boot 等框架通常完全省略 web.xml，改用 Java 配置或自动装配.

Servlet 中的响应操作

  • 通过 resp 响应对象获取输出流 PrintWriter 对象

Servlet 的创建

  • 继承 HttpServlet
  • 继承 HttpServlet 的父类 GenericServlet
  • GenericServlet 类重写了 service 方法，根据请求方法的不同，将请求分发给 doGet 和 doPost 方法
  • 继承自 Servlet 方法需要重写init()，service()，getServletConfig()，getServletInfo()，destroy()5 个方法，展现了 servlet 对象的生命周期。第一次访问时创建对象，初始化对象，请求一次就会执行一次service()方法，在 Tomcat 关闭时会执行destroy()，
  • 多个 HTTP 请求到达 Servlet 容器产生一个实例，多线程交给service()处理，单例多线程

Servlet Config 对象

  • ServletConfig 对象，获取属于 Servlet 对象的配置项。通过 web.xml 中添加标签，在doGet()或者doPost()方法中都可以创建 ServletConfig 对象来获取中的配置信息

重定向和请求转发

请求转发是一种服务器内部的跳转机制，我们知道，重定向会使得浏览器去重新请求一个页面，而请求转发则是服务器内部进行跳转，它的目的是，直接将本次请求转发给其他 Servlet 进行处理，并由其他 Servlet 来返回结果，因此它是在进行内部的转发。

请求转发可以携带数据。

  • 重定向是两次请求，请求转发是一次请求
  • 请求转发地址栏不会发生改变， 重定向地址栏会发生改变
  • 请求转发可以共享请求参数 ，重定向之后，就获取不了共享参数了
  • 请求转发只能转发给内部的 Servlet

ServletContext 对象

ServletContext 全局唯一，它是属于整个 Web 应用程序的，我们可以通过getServletContext()来获取到此对象。

ServletContext 对象，可以用来获取 web.xml 中的全局配置项，使用方法和 ServletConfig 对象一样。
