《人月神话》笔记1

发表于 2025-03-13

面试的时候经常被问自己对加班的看法是什么？
记得某一次看到别人推荐这本书，讲了一些软件开发的困境，没想到大头还是项目管理，想起来第一份工作的项目管理内容也很多，早上开会，晚上开会，觉得很烦。后来去翻看了才理解了一些工作中的项目管理。

人月神话

这本书的名字是一种项目开发的误区，把（1人/月）当成一个简单的衡量单位，1个人工作2个月和2个人工作一个月是等效的，可以换算的。例如预估12人/月的开发项目，原本有3个人，只需要开发4个月就能完成，但是中间发现无法按时完成，准备增加人手的时候，简单的把后面进来的人按照（人/月）来换算而没有考虑人员之间的沟通，会导致项目更加落后。当然还有任务的分配，新人员的培训，新的系统测试等等因素。不断加人只会导致项目像焦油坑一样淹死大型动物，很形象的比喻。
那么最好的解决办法是什么？不添加任何人手，让现有的人手加班，同时保持高效率加班。
这样客观来看的话，加班就很合理了。

沟通的负担

影响项目进度的最大因素是人员之间的沟通，确实，人越多结构越复杂，沟通的量会指数级增长。
三种沟通途径：私下沟通，会议，文档。
由此说明了文档的重要性，良好，实时更新的文档和文档结构确实可以很有效的减少沟通时间成本。
团队的组织结构也是为了较少必要的交流和沟通。

安排

书中将软件任务的进度是这样安排的：
1/3 计划
1/6 编码
1/4 构件测试和早期系统测试
1/4 系统测试。所有的构建已完成
这样的安排狠狠强调了计划和测试的重要性。
由于先后开发顺序的影响，系统测试和单元测试与项目进度牵扯的最深，而且它的时间还受到bug数量和难以捕捉的程度。

贯彻执行与测试

为了保证贯彻执行这一原则，测试小组很有必要，因为测试总会发现一些没有贯彻执行，设计决策没有准确实现或者理解正确的地方。

概念完整性

概念完整性是系统设计中最重要的概念。
这个应该是由于软件开发的概念功能多样，导致功能和理解上的复杂程度的比值才是系统设计的最终测试标准。
为了促进开发和测试的明确性。

电子电路的基本器件

发表于 2025-03-13

半导体

纯硅：电的不良导体。
N型硅：添加第五组元素（例如磷）到硅中，得到一种许多未被束缚在共价键中的自由电子的晶体。
P型硅：添加第三组元素（例如铝）到硅中，而是产生的晶体在存在大量电子空额，我们称之为空穴。

二极管

如果用一个电池使二极管的P区相对于N区为正，正向偏置，电流通过。
如果二极管的N区相对于P区为正，反向偏置，几乎没有电流。
二极管的伏安特性：电流在电压达到0.6V左右时迅速上升，因此在毫安尺度下硅二极管有一个0.6V-0.7V的阈值。

三极管(BJT)

NPN型；箭头B指向E
PNP型：箭头E指向B
原理：基极(B)小电流控制集电极(C)和发射极(E)大电流。

MOS管

原理：漏源极进出。MOSFET的工作类似源极和漏极之间的一个开关，当栅极(G)电压超过阙值时导电沟道形成，开关开通。
金属层：金属层被氧化物层（厚度不成比例）分割开来，这样就确保了金属层不会与该元件的其他部分接触。
SR模型：MOSFET 形成的导电 N 沟道并非理想导体，具有一定的电阻R。同时注意到该电阻与沟道的几何尺寸有关。设沟道长度（栅极长度）为 L，沟道宽度为 W。则电阻正比于L/W。

CMOS管

概述：反相器，将输入电平取反，通过NMOS管和PMOS管组合形成互补MOS。
结构：栅极连接在一起作为输入，漏极连接在一起作为输出。NMOS管接地，PMOS管接电源。
工作：输入高电平，NMOS管导通，输出低电平；输入低电平，PMOS管导通，输出高电平。
静态特性：输入信号不变化时，功耗低。

每日1题-统计最大元素出现至少K次的子数组

发表于 2025-03-12

题目描述

给你一个整数数组 nums 和一个正整数 k 。
请你统计有多少满足「 nums 中的最大元素」至少出现 k 次的子数组，并返回满足这一条件的子数组的数目。
子数组是数组中的一个连续元素序列。

题解

此题是典型的滑动窗口。
首先找到nums中的最大元素，然后滑动窗口保证nums中最大元素超过k个。
例如[1,3,2,3,3],遍历到第二个3的时候，cnt_mx等于k了，移动左端点直到区间内最大元素个数不超过k，此时，左端点位于第一个2处，left=2，此时[1,3,2,3][3,2,3]都满足条件，把答案增加left。当右端点移动到末尾元素时，左端点也移动到末尾元素，此时[1,3,2,3,3][3,2,3,3][2,3,3][3,3]都满足条件，答案增加left=4。

《Java核心技术》笔记6-泛型和集合

发表于 2025-03-10

泛型

类型参数的好处

在泛型之前，ArrayList维护一个Object引用的数组。

获取值时需要强制类型转换，也没有错误检查。

为了解决以上问题，泛型带来了易读性和安全性。

泛型本质

本质是一种语法糖，编译器最后会擦除类型参数，替换为限定类型，如果没有限定类型就会替换为Object，然后编译器会插入强制类型转换，由于类型擦除，我们不能用基本类型实例化类型参数。

Java泛型与C++模板有很大本质区别，C++会为每个模板的实例化产生不同的类型，模板代码膨胀。Java采用的是类型擦除。

集合

集合接口与实现分离

例如队列就有两种实现方式，一种是循环数组，一种是链表，接口指出队列的一些通用操作，例如尾部添加元素，队列头部删除元素。
分离实现的好处是，需要更改队列的实现时只需要在构造集合对象时更改具体使用的类。
Java类库中，集合类的基本接口是Collection接口，实现了一些基础方法例如add。
集合类还有一个迭代器接口，通过反复调用接口中的next方法逐个访问结合中元素，与C++的迭代器有一定区别，C++的迭代器根据数组索引建立模型，而Java的迭代器的查找操作和位置紧密耦合。查找一个元素的唯一方法时调用next，迭代器的位置会随之变化。删除元素之前也需要调用next方法越过这个元素。

集合的接口

集合提供了大量接口，基础的有Collection和Map。还有List，Set，Queue，Map，迭代器等等。

List列表

  • ArrayList数组列表
  • LinkList链表，双相链表，使用链表的唯一理由是尽可能减少在列表中间插入和删除元素的开销。

Set集合

不支持重复元素，不支持下标访问。

  • 散列集HashSet
  • 树集TreeSet，采用红黑树，实现有序集合

Queue队列和Deque双端队列

  • Deque接口，ArrayDeque和LinkedList类都实现了这个接口

优先队列

允许高效删除最小元素的一个集合

  • PriorityQueue

异常

分类

Java的异常都继承自Throwable。下一层分为两类，Error和Exception。Exception又分为两类，RuntimeException和I/OException。
非检查型异常：

  • Error类错误描述Java运行时系统内部错误和资源耗尽错误。应用程序不应该抛出这种异常。
  • RuntimeException类错误，由编程错误导致的异常，其包括错误的类型转换，数组访问越界，访问null指针。

检查型异常：

  • 所有其他异常，例如I/O错误，试图打开一个不存在的文件，试图超越文件末尾读取数据。

    C++中exception类下有两个子类，一是logic_error和runtime_error。但是logic_error相当于Java中RuntimeException，表示程序中的逻辑错误，runtime_error相当于Java中非RuntimeException类型的异常，表示由于不可预测的原因引发的检查型异常。

声明检查型异常

总之，检查型异常需要声明，而非检查型异常从一开始就需要避免。

C++异常与Java异常有很大不同，C++的throw在运行时执行，而Java在编译时执行。Java要求必须给一个方法声明所有可能的检查型异常。

如何抛出异常

声明异常之后需要在方法内部抛出异常，只需要使用throw new EOFexception(),new后面是声明的异常。相当于

在Java中，只能抛出Throwable子类的对象，而C++可以抛出任何类型的值

捕获异常

如果try语句中出现了catch中的异常，那么会跳过try语句中剩余代码，直接执行catch中代码；
如果没有出现异常，会跳过catch子句。
Java也支持捕获多个异常，只需要增加catch子句就行。
Java支持一个新的关键词，finally，允许不管是否有异常被捕获，finally子句中的代码都会执行。

在Java中，没有catch(…)这样的代码存在，因为所有异常派生于Throwable这个类。

捕获和抛出异常的原则是尽量捕获你知道如何处理的异常；继续传递那些不知道如何处理的异常，同时添加到throws列表里。

每日1题-每一个查询的最大美丽值

发表于 2025-03-09

题目描述

给你一个二维整数数组 items ，其中 items[i] = [pricei, beautyi] 分别表示每一个物品的价格和美丽值 。
同时给你一个下标从 0 开始的整数数组 queries 。对于每个查询 queries[j] ，你想求出价格小于等于 queries[j] 的物品中，最大的美丽值是多少。如果不存在符合条件的物品，那么查询的结果为 0 。
请你返回一个长度与 queries 相同的数组 answer，其中 answer[j]是第 j 个查询的答案。

题解

最简单直观的做法就是对于每次查询，遍历得到最大美丽值。

这种做法会超时。所以需要优化时间复杂度。
对二维数组的价格进行排序，对查询进行排序。对于每个查询，定义一个指针i指向物品数组，每次移动指针都更新当前的最大美丽值。
每次移动指针直到当前价格小于查询值。
